What This App Does
ConfigFlow is a feature flag + config management service. It lets you:

Store key/value configs with a rollout percentage (0–100%)
Serve those configs to a demo endpoint that decides if a feature is ON/OFF per user
Keep a full audit trail of every change
Data Model
How Rollout Works
When you call /api/demo?userId=xyz, it:

Looks up key feature.newCheckout in the DB
Takes rolloutPercent (e.g. 10)
Hashes userId + key with CRC32 → gets a bucket 0–99
If bucket < rolloutPercent → feature is ON for that user
This means the same userId always gets the same result (deterministic).

# 1. Create flag at 0% — nobody gets it
curl -X POST http://localhost:8080/admin/config/feature.newCheckout `
  -H "Content-Type: application/json" `
  -d '{"value":"true","rolloutPercent":0,"updatedBy":"tester"}'

curl "http://localhost:8080/api/demo?userId=alice"
# → "enabled": false

# 2. Roll out to 100% — everyone gets it
curl -X POST http://localhost:8080/admin/config/feature.newCheckout `
  -H "Content-Type: application/json" `
  -d '{"value":"true","rolloutPercent":100,"updatedBy":"tester"}'

curl "http://localhost:8080/api/demo?userId=alice"
# → "enabled": true

# 3. Check audit — should show both changes
curl http://localhost:8080/admin/audit
